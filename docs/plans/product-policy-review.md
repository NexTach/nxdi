# NXDI 상품정책 확정 및 잔여 확인표

[기준일] 2026/07/15

이 문서는 공개 설명문이 아니라 현재 코드와 정책의 일치 여부를 관리하는 내부 점검표다. 법률·세무 자문을 대체하지 않는다.

## 확정한 상품 규칙

| 영역 | 확정 규칙 |
|---|---|
| 사이트 의향서 | 투자·출금 의사를 전달하는 비구속적 기능이다. `수락`은 후속 연락 대상 표시일 뿐 청약 승낙, 계약, 투자자 지위, 입금의무, 원금, 분배권 또는 지급권을 만들지 않는다. |
| 의향 철회 | 별도 계약입금·실제 출금정산과 연결되기 전에는 본인이 철회할 수 있다. 연결 뒤에는 별도 계약 절차를 따른다. |
| 실제 계약 | 의향서와 별도로 계약서 식별값·버전·서명시각, 입금 거래식별값·입금시각을 남긴다. 실명·계좌·적합성·AML·제재 확인이 유효해야 계약입금 원장을 만들 수 있다. |
| 계좌 | 은행·증권계좌는 NexTach 법인 명의이며 일반 고유재산 계좌와 분리한다. 분리만으로 원금보장이나 도산절연이 되는 것은 아니다. |
| 운용편입 원금 | 수락 의향, 계약금액, 입금액, 미편입 예수금 및 재투자 대기금은 원금이 아니다. 실제 종목 매수 체결의 원화 결제금액이 투자자 자금원천에 배정된 때에만 원금이 증가한다. |
| 평가이익 | 투자자는 평가이익에 참여하지 않는다. 가격·환율 상승만으로 원금은 늘지 않고, 출금 기준액은 원래 운용편입 원금을 넘지 않는다. |
| 손실 순서 | 회사 자본이 먼저 손실을 부담한다. 정확한 표현은 `회사 자본의 선손실 부담` 또는 `회사 후순위 자본`이며, 채권의 선순위 변제권을 뜻하는 `회사가 선순위 손실흡수`는 쓰지 않는다. |
| 회사 지원 | 회사 기본 분배몫의 최대 20%를 투자자 현금상한 부족분 범위에서 이전한다. 회사의 희생을 통한 보호장치이나 목표수익·손실보전·지급보증은 아니다. |
| 운용보수 | 투자자 총귀속액의 5%. 총귀속액이 있는 달의 월말에만 산정하며 원금 차감·미납이월은 하지 않는다. |
| 현금 상한 | 투자자 운용편입 원금의 월 1.5%. 18%는 월 상한의 12배 단순 환산일 뿐 연간 누적 상한이나 목표수익률이 아니다. |
| 상한 초과분 | 회사 수익으로 돌리지 않고 투자자별 재투자 대기금으로 남긴다. 실제 매수 때만 원금으로 전환한다. |
| 분배 기준일 | 실제 매수 편입월의 다음 달부터 분배 대상이다. 월말 확정 순자산과 실제 편입·출금 원장을 사용한다. |
| 월 실분배금 | 종목별 실제 입금일의 증권사 체결환율로 즉시 원화 환전하고 외국세·증권사비용·환전비용을 공제한 순입금 원장의 합계다. 임의 월 합계와 대체환율은 금지한다. |
| 환헤지 | 실시하지 않는다. 신용·차입·마진, 공매도, 파생상품 직접투자, 레버리지·인버스 상품, 증권대차도 금지한다. |
| 지급일 | 투자자 분배금은 매월 말일 지급 예정이며 비영업일이면 다음 영업일이다. 정산자료·은행·시장 장애 시 지연될 수 있다. |
| 출금 | 출금 의향 수락과 실제 출금은 별개다. 별도 출금지시서, 최신 월말평가, 실제 매도·결제현금, 본인명의 계좌 및 은행 지급 거래식별값이 있어야 원금 차감과 지급을 기록한다. |
| 오입금·미편입 반환 | 오입금은 확인 완료 후 원칙적으로 3영업일 내 반환한다. 미편입 자금은 해당 자금원천의 미사용 잔액 한도에서 운용자산 출금과 분리해 반환한다. |
| 반올림 | 원 단위는 최대잔여법으로 개인에게 배정한다. 월 전체 산식 잔액은 투자자가 있으면 재투자금에 넣어 회사 수익으로 남기지 않는다. |

## 구현된 자동 통제

- 수락 의향은 자본원장이나 포트폴리오 총자산에 들어가지 않는다.
- 실제 매수 체결만 FIFO 자금원천 배정을 거쳐 투자자 원금을 증가시킨다.
- 종목 편집으로 수량·평단을 바꾸거나 잔고가 있는 종목을 직접 삭제할 수 없다.
- 상대 위험등급이 없는 종목의 신규 매수를 차단한다.
- USD 거래와 현금분배금에 실제 환율이 없으면 저장·계산하지 않는다.
- 화면 환율은 검증된 최근 성공값을 36시간까지만 허용하고 고정 대체값을 사용하지 않는다.
- 가격 갱신이 하나라도 실패하면 월말 평가 스냅샷을 만들지 않는다.
- 최신 확정 스냅샷이 모든 체결 이후를 포함하지 않으면 실제 출금 정산을 차단한다.
- 미편입 자금보다 큰 반환, 실제 원금보다 큰 출금, 결제현금보다 큰 지급을 차단한다.
- 종목별 현금분배금은 거래명세 식별값으로 중복을 막고, 오류는 삭제가 아닌 반대분개로 남긴다.
- 월 정산안 계산 후 현금분배금 원장이 바뀌면 확정을 차단한다.
- 월 정산 확정과 실제 은행 지급을 분리한다. 은행 지급·원천세 납부 식별값이 있을 때만 현금원장을 차감한다.
- 지급 실패는 미지급 상태와 실패사유를 남기며 투자자 귀속액을 회사 수익으로 전환하지 않는다.
- 출금계좌 번호는 AES-256-GCM으로 암호화해 저장하고 화면에는 끝 4자리만 노출한다.
- 상품설명·분배정책 버전과 동의시각을 의향서마다 저장한다.

## 운영 전에 회사가 제공해야 하는 정보

다음 값은 코드가 대신 정할 수 없다. 제공받으면 공개 문서·계약서·환경설정에 반영해야 한다.

1. NexTach의 정식 상호, 법인등록정보, 주소, 대표자, 금융투자업 인가업무 단위·인가번호·감독기관 조회 링크
2. 분리계좌를 제공하는 은행·증권사, 계좌 유형, 인출권자, 일별 대사 담당자와 상계·압류·도산 시 법률 검토 결과
3. 상품의 법적 계약형태와 투자자 범위, 모집방식, 최소 법정 교부문서 및 실제 계약서 최종본·버전
4. 세무사가 확정한 소득 구분, 원천징수의무자, 적용 세율, 재투자금 과세시점, 신고·납부 식별자료
5. 민원 담당부서, 전화, 전자우편, 처리기한과 이용 가능한 분쟁조정기관
6. 개인정보 처리방침, 보유기간, 파기기준, 개인정보 보호책임자, 암호화 키 교체·복구 절차
7. 대표자 부재·시스템 장애·인가변경·계좌동결 때 업무를 인계할 대체 운영자와 비상연락망

## 배포·운영 체크

1. 데이터베이스 마이그레이션을 적용한다.
2. `DATA_ENCRYPTION_KEY_BASE64`에 안전하게 생성한 32바이트 키를 비밀관리 시스템으로 주입한다.
3. 기존 평문 출금계좌가 있다면 `npm run db:encrypt-sensitive`를 1회 실행하고 백업까지 암호화됐는지 확인한다.
4. 실제 계약서·상품설명서·분배정책 버전을 배포본과 맞춘다.
5. 회사 내부자금만으로 계약입금 → 실제 매수 → 월말평가 → 종목별 분배금 입금 → 월정산 → 지급 → 재투자 → 출금의 전 과정을 리허설한다.
6. 은행·증권사 잔고와 현금·증권·세금 원장의 일별 대사 절차를 별도 운영통제로 둔다.
7. 세율과 법적 표시사항이 확정되기 전에는 실제 투자계약·입금 또는 투자자 지급을 진행하지 않는다.

기존 투자자 마이그레이션은 현재 범위에서 제외하되, 실제 기존 거래가 생기면 익명화 원장 대사 없이는 새 원장에 임의 편입하지 않는다.
